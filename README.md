# FitBuddy — AI Fitness & Nutrition Coach

FitBuddy is a gamified, fully responsive web application designed to act as an AI-powered fitness and nutrition coach. The app is optimized for both desktop and mobile layouts, ensuring an outstanding user experience on any device.

## 🚀 Features

*   **Custom Onboarding Wizard:** Real-time BMI calculations, TDEE estimations (Mifflin-St Jeor), and macro allocation targets customized to user goals.
*   **AI Coach (watsonx.ai):** Direct chat with a coaching assistant powered by IBM Granite models. The coach adapts to user profiles, provides recipes tailored to preferred cuisines/diets, and handles special inputs like cravings or stress.
*   **Progress Dashboard:** Interactive loggers for meals, exercises, water intake, sleep, and mood.
*   **Gamification & Rewards:** Leveling system where logged activities earn XP. Reaching new levels unlocks customized healthy recipes.
*   **Zen Play Zone:** A mini-games section and a guided breathing tool for days when the user reports high stress or fatigue.
*   **Cloud Architecture:** Serverless backend functions with persistent database storage.

## 🛠️ Tech Stack

*   **Frontend:** Vanilla HTML5, CSS3 (with responsive breakpoints and modern layout tokens), and Vanilla ES Modules.
*   **Backend:** Vercel Serverless Functions (Node.js).
*   **Database:** Vercel KV (Serverless Redis) for user credentials and state serialization.
*   **AI Integration:** IBM Cloud watsonx.ai APIs.

## ⚙️ Environment Variables

To deploy or run the project, the following environment variables are required:

| Variable | Description | Source |
|---|---|---|
| `DB_TOKEN` | A secure string used to validate frontend sessions. | Generate any random string |
| `REDIS_URL` | The Vercel KV database connection string. | Auto-generated by Vercel KV |
| `IBM_API_KEY` | Your IBM Cloud IAM API Key. | IBM Cloud Console |
| `IBM_PROJECT_ID` | Your watsonx.ai project ID. | watsonx.ai Console |
| `IBM_REGION` | The region of your IBM Cloud service (e.g., `us-south`). | IBM Cloud Console |
| `IBM_MODEL_ID` | The Granite model ID (defaults to `ibm/granite-3-8b-instruct`). | IBM Cloud Console |

## 📦 Deployment & Setup

### Vercel Deployment

1.  Connect your GitHub repository to Vercel.
2.  Navigate to the **Storage** tab in your Vercel project dashboard and click **Create Database**.
3.  Choose **Redis (Official Redis for Vercel)**. Select the free tier (choose "None" for high-availability) and link it to your project.
4.  Navigate to **Settings → Environment Variables** in Vercel and add `DB_TOKEN`, `IBM_API_KEY`, `IBM_PROJECT_ID`, and `IBM_REGION`.
5.  Trigger a new deployment under the **Deployments** tab by clicking **Redeploy**.

### Local Development

To run the serverless function emulator locally, install the Vercel CLI and run:

```bash
# Install dependencies
npm install

# Run the local serverless dev server
npm run dev
```
